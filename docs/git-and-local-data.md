# Git とローカルデータ

ローカル開発で生成されるファイルと、リポジトリに含めないものの整理です。[`.gitignore`](../.gitignore) が実体の一覧です。

## 環境変数

- **`.env`** … 秘密情報を含むためコミットしません。ひな形は **`.env.example`** のみ追跡します（`!.env.example`）。
- 新しい環境変数をコードに追加したら、**`.env.example` にキーと短いコメント**を足し、実値は入れないでください。

## Next.js / Node

- **`node_modules`** … 依存は `package-lock.json` から復元します。
- **`.next/`** … ビルド成果物。`npm run dev` / `npm run build` で再生成します。
- **`*.tsbuildinfo`** … インクリメンタル用。再生成されます。

## 画像（ローカル開発）

Cloudinary の環境変数が無いとき、アップロード画像は **`public/uploads/`** に保存されます。このディレクトリは **`.gitignore`** 対象です。各自のマシン専用のためコミット不要です。

## データベース

アプリは **PostgreSQL**（Neon 等）を前提にしています。`.gitignore` に SQLite 用のパターン（`*.db` など）が残っている場合は、過去のテンプレート由来で、**使っていなければファイルは生成されません**。

接続先は常に **`.env` の `DATABASE_URL`** と [`prisma.config.ts`](../prisma.config.ts) を一致させてください。

## シードと本番 DB

`npm run db:seed` は **既存ユーザーを削除してから**サンプルを投入します。**本番や実ユーザーのいる DB では実行しない**でください。手順の詳細は [README の「サンプルデータ（シード）」節](../README.md#サンプルデータシード) を参照してください。

## その他

- **`.vercel`** … Vercel CLI のリンク情報。
- **`*.pem`** … 証明書類の誤コミット防止。
