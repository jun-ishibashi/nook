## 1. 北極星（一言で）

**「部屋と家具の WEAR」** — 若者の**新しいおしゃれ**に出会い、**自分の部屋で発信**できる。リアルな部屋と、家具・雑貨の**どこで買ったか**が辿れる体験で、気分から行動までつなぐアプリ。

- **主役**は **発見**（みんなのおしゃれ・ムードに出会うこと）と **発信**（自分の部屋を恥ずかしくなく見せること）。**購入先まで辿れる便利さ**は、その信頼と行動を支える（[§2](#2-プロダクト原則優先順位) の原則 3）。
- **WEAR** が「今日のコーデ × アイテムの購入先」をつないだように、**NOOK** は「この部屋 × 家具・雑貨の購入先」をつなぐ。
- 目的は **カタログで全部売ること** ではなく、**インスピレーションと購買のあいだを短くすること**。
- **シンプル**であることで、その薄さと速さを守る（コピー・画面・機能は **足しすぎない**。[§2](#2-プロダクト原則優先順位) の原則 4）。

### 1.1 WEAR を引き合いにした捉え方（比喩）


| WEAR に近いところ | NOOK                 |
| ----------- | -------------------- |
| コーデ写真が主役    | 部屋・インテ写真が主役（**発見・発信**の中心）   |
| タイムラインで発見   | フィード・検索・ムード分類で**おしゃれに出会う**   |
| アイテムにリンク    | 家具・雑貨に名前・購入 URL（＋メモ）で**辿れる** |
| 保存・いいね      | 保存・いいね・**欲しい**       |

| WEAR と違うところ  | 理由                                 |
| ------------ | ---------------------------------- |
| 部屋は一点物になりやすい | **文脈（間取い・賃貸等）** を厚くできる             |
| 購入は外部 EC が基本 | **リンクの透明性** と **信頼**（自社在庫に引きずられない） |

## 価値提案と Jobs to be done

> [§1 北極星](#1-北極星一言で) の直後に置く **戦略ブロック**。誰の・どんな「仕事」を・なぜ NOOK なら依頼されるかを短く固定し、ピッチ・採用・優先度議論の共通言語にする。

### 解きたい摩擦（課題の置き方）

- **発見**：好みの**若者のおしゃれ・ムード**に**すぐ出会えない**（SNS は流れやすい、検索はノイズや粒度が合わない、同じ画の連続で飽きる、など）。
- **発信**：自分の部屋を**恥ずかしくなく載せ続けたい**が、フォームが重い・用語が難しい・トーンが合わないと **UGC が続かない**。
- **購入までの辺**：きれいな部屋は見つかっても「その照明どこ？」が曖昧で、**購入までの経路が長い**。画像のブックマークは溜まっても**行動に繋がりにくい**。
- **信頼と気分の両立**：アフィリエイトや編集記事ばかりだと **本当に自分の部屋か／在庫があるか** が不透明になり、若年層ほど **恥ずかしさ・古さ** を感じやすい。

### NOOK が担う「仕事」（Jobs）


| Job（ユーザーが成し遂げたいこと）      | NOOK の答え（要約）                                       |
| ----------------------- | -------------------------------------------------- |
| **好みの「リアルな部屋」に早く出会いたい** | フィード・検索・分類で**おしゃれ・ムード**に辿り着く（[原則 1](#2-プロダクト原則優先順位)）   |
| **恥ずかしくないトーンで、自分の部屋を載せたい** | 敬体短文・コンクリ寄り UI・内部用語を表に出さない。載せる摩擦を軽く（[原則 2](#2-プロダクト原則優先順位)） |
| **その部屋の家具を、曖昧さなく調べたい**  | 写真と **購入先 URL を同じ体験で**（[原則 3](#2-プロダクト原則優先順位)）     |
| **あとで買う・似た雰囲気を探したい**    | 保存・欲しい・同じ URL／ショップへの導線（過剰通知はしない）                   |


### 提供価値の一文（ピッチ用）

**若者のおしゃれに出会い、自分の部屋で発信できる。** **賃貸リアル**のムードが中心で、家具・雑貨は**どこで買えるかまで同じ体験で**辿れる。EC でも百科でもなく、**あいだを短くするレイヤー**。

## 市場における立ち位置と差別化

> 価値提案に続く **戦略ブロック**。**ウェッジ**（どこで戦うか）と **モート**（時間が経つと効く資産）、**GTM の含意**をここで言語化する。競合を貶すのではなく **陣取り** を固定する。

**この節の流れ**：ジャンルマッピング → [RoomClip との整理](#roomclip-との整理) → [差別化の軸](#差別化の軸プロダクトgtm-共通) → [モート](#中長期で厚くしたいモート資産のイメージ) → [追い風・リスク](#マーケットの変化と追い風リスク) → [GTM](#go-to-market-の含意方針レベル) → [マネタイズ](#マネタイズの設計方針中長期)

### 近いジャンルのマッピング（簡易）


| ジャンル                        | ユーザーにとっての強み | NOOK との距離（ざっくり）                                       |
| --------------------------- | ----------- | ----------------------------------------------------- |
| **汎用 SNS**（例：Instagram）     | 発見・フォロー文化   | 購入導線が非構造。**部屋×URL の一貫性**は弱くなりがち                       |
| **画像ブックマーク**（例：Pinterest 系） | ムードボード・収集   | 「この投稿のこのアイテムの URL」まで **粒度が粗くなりがち**                    |
| **インテリアメディア・まとめ**           | 編集の安心感      | **自分ごとの賃貸リアル**とズレやすい。更新コスト・広告とトレードオフ                  |
| **家具・インテ EC・アプリ**           | 在庫・決済まで完結   | **インスピレーションの気分・ライフ文脈**が二の次になりやすい（[§7](#7-やらないこと) と親和） |
| **掲示板・Q&A 型コミュニティ**         | 情報の深さ       | **写真主役**・ノイズ抑制が難しくなりがち                                |
| **部屋・インテリアの投稿型プラットフォーム**（例：[RoomClip](https://roomclip.jp/)） | 部屋写真の蓄積・発見・コミュニティ・案件・編集コンテンツなど **複合的な強み** | **家具ごとの購入 URL** を投稿単位の価値の芯に据え、**若年・軽い操作・透明な外部遷移**で陣取りを分ける（[RoomClip との整理](#roomclip-との整理)） |


NOOK は上記の **いずれか一つに取って代わる** より、**「おしゃれに出会う → 載せる → 欲しい → 外部で買う」** の **あいだ専用の薄くて速いレイヤー** を取りにいく。**主価値は発見と発信**で、購入先の一貫性はその**信頼と行動**を支える。

### RoomClip との整理

> [RoomClip](https://roomclip.jp/) は日本で **部屋・インテリアのインスピレーション** を集める **代表例** として比較する（公式提携・機能仕様の記述ではない。RoomClip 側の進化は都度アップデートが必要）。

ユーザーは **同じ「きれいな部屋を知りたい」** で RoomClip と NOOK を **併用しうる**。差別化は **優劣の断定**ではなく、**捨てるもの・一番になりたい軸** を NOOK 側で固定する。

| 観点 | RoomClip に近いこと（一般論） | NOOK が厚く置く差分 |
| --- | --- | --- |
| **価値の単位** | 部屋写真・ノウハウ・コミュニティ・案件など **多面** | **発見・発信**（若者のおしゃれ・ムード）を主にしつつ、**1 部屋の中で** 家具・雑貨行と **購入 URL をセット**（[原則 3](#2-プロダクト原則優先順位)）。リンクなし投稿を **薄い** と捉え、載せる導線で自然に寄せる |
| **コミュニティ・機能幅** | 交換会・プロ・キャンペーン等、**厚いエコシステム**になりやすい | **コミュニティの網羅**を最初から狙わない。**発見から保存・欲しいまでの短いループ**を優先（[§3](#3-迷ったときの判断基準)・[§5](#5-ui--ux-基盤)） |
| **購買・収益との関係** | 商品・案件・メディアと **複合しやすい** | **外部ショップへの遷移を隠さない**。自社在庫 EC 完結（[§7](#7-やらないこと)）に寄せず、**透明性＝ブランド** を維持 |
| **トーン・ペルソナ** | 幅広い層・ライフステージ | **若年・賃貸リアル・特定ムード**（[§6](#6-ターゲットと世界観)）にコピーとフィードを寄せ、**万能インテアプリ**との正面衝突を避ける |

**運用上の含意**：RoomClip で得た関心を **否定せず**、「**どこで買うかまで一気に辿りたい**」層向けに NOOK の体験を尖らせる。比較表現は **事実ベース**に留め、他社を貶すマーケは [§6.3](#63-若者向けおしゃれコンクリート感) のトーン（煽り・セール調 NG）に反しやすい。

### 差別化の軸（プロダクト・GTM 共通）

1. **コンテンツ単位の一貫性**
  **主役は発見・発信**（おしゃれ・ムードの体験）。そのうえで **1 部屋の中で** 家具行と購入 URL がセットし、信頼と行動に繋がる。スクショ投稿やリンクなし投稿を **価値が薄い** と捉え、載せる導線で自然に寄せる（説教ではなく UI と空状態で）。
2. **透明性＝ブランド**
  外部ショップへの遷移を隠さない。**自社在庫 EC 化**（§7）に寄せないことで、若年層の **アフィリエイト疲れ・不信** と差別化しうる。
3. **リアルと文脈**
  賃貸・ワンルーム・「住んでそう」を **排除せず中心に**。サンプルルーム・AI 生成だらけのフィードとの **対比** になりうる（過激な煽りはしない）。
4. **操作と情報の軽さ・シンプル**
  余計な画面枠・バッジ・長文を増やさない（[§3](#3-迷ったときの判断基準)・[§5](#5-ui--ux-基盤)）。**スマホでの連続スクロール**に最適化。**シンプル**＝迷ったら **削る**が先（[§2](#2-プロダクト原則優先順位) 原則 4）。

### 中長期で厚くしたい「モート」（資産のイメージ）

時間が経つほど効くものだけを列挙する（確約ではない）。


| 資産                            | なぜ効くか                                                                               |
| ----------------------------- | ----------------------------------------------------------------------------------- |
| **部屋 → アイテム → URL／ドメイン** のグラフ | 同じ購入先・同じショップを辿る **発見のショートカット** が増える（ショップ別の見せ方などで具体化） |
| **信頼できる UGC の供給**             | 載せる摩擦が低く、**恥ずかしくない**とクリエイターが残りやすい（[原則 2](#2-プロダクト原則優先順位)）                           |
| **トーンと審美の一貫性**                | 「閉店感のない・若く見える」フィード体験が **コピーしにくい文化層** になりうる                                          |
| **検索・分類の語彙**                   | 世界観に合った **寄り道語彙**が蓄積し、コピー・発見の表現と一貫する                      |


### マーケットの変化と追い風・リスク


| 変化                  | NOOK にとって                                              |
| ------------------- | ------------------------------------------------------ |
| 賃貸での自己表現・リモートワーク    | **「自分の部屋」** の関心が継続しやすい                                 |
| D2C・越境 EC の普及       | **「どこのサイトか」** が明示される価値が上がる                             |
| アフィリエイト・ステルス広告への警戒  | **リンク透明**・短文トーンが相対的に有利になりうる                            |
| AI による画像・記事の大量生成    | **リアルな部屋・文脈メモ** の希少性が上がる一方、**偽・誇張**との差別化が課題（信頼設計がより重要） |
| 大型プラットフォームのアルゴリズム変化 | **独自グラフ・フォロー・検索** を持つほど依存リスクを下げられる                     |


### Go-to-market の含意（方針レベル）

- **最初は狭く濃く**：インテ・賃貸リアルに共感する層で **投稿の質と再訪** を優先。万能インテアプリを名乗らない。
- **クリエイター・拡散**：「サンプルルーム風の完璧さ」より **住んでいそうなリアル**（[§6](#6-ターゲットと世界観)）を称賛するコミュニケーションがブレない。
- **B2B（ブランド・小売）** は後段でもよいが、**§7 に反する収益**（在庫主導・ランキング煽り）に寄せない契約設計にする。

### マネタイズの設計方針（中長期）

> **ロードマップや収益の確約ではない。** 収益機能・提携・契約を検討するときの **ガードレールと候補レーン** を固定する。手数料率・契約書・課金基盤の具体は本書のスコープ外（運用・法務は別途）。

**結論**：NOOK は **マネタイズ可能** と考える。差別化の芯は **若者のおしゃれの発見と自分の発信**に加え、[差別化の軸](#差別化の軸プロダクトgtm-共通) の **写真×購入先の一貫性** と **外部遷移の透明性** である。**在庫・決済完結への寄せ**や**リンク・広告のステルス化**はブランドと両立しにくい。一方、**明示とユーザー主導を保った購買誘導**、**B2B（ブランド・小売）**、**クリエイター／ショップ向けの付加価値**は、上記 GTM の B2B 記述と整合しやすい。

#### 前提（非交渉）

収益の議論は次を **上書きしない**。

- [§2 プロダクト原則](#2-プロダクト原則優先順位)（特に **原則 1・2**（発見・発信）と **原則 3**（写真と「どこで買えるか」のセット）。収益 UI も **原則 4（シンプル）** でノイズ化しない）
- [§7](#7-やらないこと)（総合 EC 代替・百科化・過度ゲーミフィ・排他）
- **透明性＝ブランド**（[差別化の軸](#差別化の軸プロダクトgtm-共通) 2）：外部ショップへの遷移を隠さない
- [§6.3](#63-若者向けおしゃれコンクリート感)（煽り・セール調・チラシバナーだらけに寄せない）

#### 相性の高いレーン（候補）

確約ではない。各レーンで **守る／避ける** を先に決めてから具体化する。

| レーン | 守るもの（例） | 避けるもの（例） |
| --- | --- | --- |
| **購買連動の紹介**（アフィリエイト等） | 種別・関係性の**明示**、短文。ユーザーが「有料紹介」と誤解しない UI。導入時は**表示・法務は専門家確認** | ステルス広告、リンク先のすり替え、実質的な「公式」偽装 |
| **B2B（ブランド・小売）** | **スポンサー／公式**のラベル、**ショップ検証**など信頼を足す設計。契約は在庫主導・ランキング煽りに寄せない（上記 GTM と同型） | フィード順位の**不透明な有料操作**、§6.3 に反するセールコピー強制 |
| **クリエイター／プロ向け付加価値** | 分析、下書き同期、投稿支援など **[§8](#8-将来検討できる機能候補) の候補**と接続しうる。**原則 2**（発信・載せる摩擦）を下げる方向 | 投稿の「見え方」を損なう押し付け、バッジ・ランキング至上（§7） |
| **データ／インサイト**（集約レポート等） | **匿名・集約**、利用目的の説明、オプトの考え方 | 個人の行動の売買、利用者不信に直結する見せ方 |

#### 相性が低い・要慎重なレーン

- **自社在庫 EC 完結**を主目的にした手数料・在庫主導（§7・差別化と衝突しやすい）
- **ステルス広告**・**実質ステマ**に近い案件（透明性の否定）
- **課金でフィードや検索の順位を説明不能にする**（信頼と原則 1 の発見を損なう）
- **過度なゲーミフィケーション**とセットの収益（§7）

#### 順序（GTM と収益の時間軸）

[Go-to-market の含意](#go-to-market-の含意方針レベル) の **最初は狭く濃く** と揃える。[モート](#中長期で厚くしたいモート資産のイメージ)（部屋→アイテム→URL のグラフ、UGC、トーン）が薄い段階での **過剰な収益化**は、若年層の不信（[マーケットの変化と追い風・リスク](#マーケットの変化と追い風リスク)）と相まって取り返しにくい。**供給と再訪が先、収益実験は後**でもよい、という順序をデフォルトにする。

#### 収益・提携の判断チェックリスト（§3 補助）

収益機能・広告枠・契約のレビューで、次を **はい** にできなければ再設計または見送り。

| 問い | はいなら進めやすい | いいえなら再考 |
| --- | --- | --- |
| **[原則 1・2](#2-プロダクト原則優先順位)**（発見・発信）を弱めないか？ | おしゃれ・ムードの発見と、載せ続けられる体験が主役のまま | フィードが購入導線だらけ、載せる・見るの気分が後景になる |
| **[原則 3](#2-プロダクト原則優先順位)**（写真×購入先のセット）を弱めないか？ | 家具行・外部遷移が明確。①②を薄めない | 購入導線が曖昧、リンクが従、または URL 優先で写真・ムードが負ける |
| **透明性**（誰のための表示か）がユーザーに伝わるか？ | ラベル・説明で十分 | ステルスに見える、説明を隠す |
| **[§7](#7-やらないこと)** に触れていないか？ | EC 完結主目的・百科肥大・バッジ地獄・排他ではない | 在庫決済が主、煽りランキング、特定スタイル沈め |
| **[§6.3](#63-若者向けおしゃれコンクリート感)**（煽り・セール調）に寄せていないか？ | トーンは短文・落ち着き | セールだらけ通知・バナー、閉店感 |
| **[§5](#5-ui--ux-基盤)**（写真主役・操作の軽さ）を損なわないか？ | コンテンツ層を圧迫しない | 広告帯が一覧・詳細を食う |
| **[原則 4](#2-プロダクト原則優先順位)**（シンプル）を守れるか？ | 説明・枠・導線が必要最小限 | 収益のための情報や操作が積み上がる |

## 2. プロダクト原則（優先順位）

番号が若いほど **譲りにくい**。[§3](#3-迷ったときの判断基準) のチェック・[§7](#7-やらないこと) のスコープ外は、すべてこの順序の下に置く。

**原則 1 と 2** は **閲覧（発見）と発信**を同格の主役とみなす。供給と需要のどちらか一方だけを犠牲にしないトレードオフは **§3 で切る**。

1. **若者のおしゃれ・ムード・リアルな部屋を発見できる**こと。フィード・検索・分類で、**好みにすぐ辿り着ける**速さを含む。
  → *操作の段数・認知負荷より、おしゃれに出会うまでの到達を優先しがち。*
2. **自分のおしゃれを発信し、続けられる**こと。恥ずかしくないトーン（コピー・UI・既定テーマ）。載せる摩擦は軽く。
  → *「整っているが軽い」敬体短文とコンクリ寄りの UI。フォームの長さ、用語、保存の安心感を意識する。*
3. **写真と「どこで買えるか」がセット**であること。発見・発信の**主役を弱めない**範囲で、信頼と行動に繋げる。
  → *一覧も詳細も、家具行も、外部遷移の透明性もこの下に置く。URL 優先で写真・ムードが負けないようにする。*
4. **シンプルであること。** コピー・画面・機能は **足しすぎない**。迷ったら **削る**を先に検討する。
  → *原則 1〜3 を満たすための追加だけに留める。説教・ダッシュボード化・「とりあえず置ける」枠の積み上げは避ける。*

## 3. 迷ったときの判断基準

[§2 プロダクト原則](#2-プロダクト原則優先順位) と [§7](#7-やらないこと) を踏まえ、トレードオフで迷ったときの **チェックリスト**。コピーは [§4](#4-ブランドトーンと表記)、操作の見た目は [§5](#5-ui--ux-基盤) と対で使う。収益・提携・有料枠の議論では [マネタイズの設計方針（中長期）](#マネタイズの設計方針中長期) の判断表も併用する。

### 3.1 価値とスコープ


| 問い                                   | はいなら進めやすい                | いいえなら再考                        |
| ------------------------------------ | ------------------------ | ------------------------------ |
| **原則 1** を強めるか、損なわないか？               | 若者のおしゃれ・ムードへの到達が速い／気分が立つ | 発見経路が増えすぎて迷う、操作帯や装飾が増えて画が負ける   |
| **原則 2** を強めるか、損なわないか？               | 載せる・続ける体験が軽く、トーンが若く恥ずかしくない | フォーム肥大、用語難、古い・押しつけの UI        |
| **原則 3** を強めるか、損なわないか？               | 写真・購入先のセットが明確。**①②を薄めない** | 購入だけが前に出る、またはリンクが曖昧で信頼が落ちる    |
| **原則 4**（[§2](#2-プロダクト原則優先順位)）を強めるか、損なわないか？ | 説明・項目・導線が **必要最小限**、迷いが減る | 機能・文言・UI 枠が積み上がり、一覧や詳細が雑然とする |
| **やらないこと**（[§7](#7-やらないこと)）に触れていないか？ | EC 完結・百科化・強いゲーミフィ・排他ではない | 在庫決済主目的、網羅説明の肥大、バッジ地獄、特定スタイル排除 |


### 3.2 トーン（コピー）


| 基準          | 寄せる方向                                              | 避ける方向                  |
| ----------- | -------------------------------------------------- | ---------------------- |
| 文体          | **です・ます**、**短文**                                   | 常体の一行投げ、説明書調の長い敬語の連発   |
| 人称          | **みんな**、**自分の部屋**、名詞主語                             | **あなた**の多用、子ども向けの過剰な砕け |
| 呼び方（ユーザー向け） | **部屋**（一覧・詳細の単位）、行為の補足は **写真を載せる**、ナビは **載せる**、ブックマークは **保存**、リストは **欲しい** | ユーザー向けで「ブックマーク」「内部用語」  |
| 法務・規約       | 本文は **正確さ優先**                                      | 条文全体を口語だけにする           |


### 3.3 UI / UX（見た目と操作）


| 基準             | 寄せる方向                                                                | 避ける方向               |
| -------------- | -------------------------------------------------------------------- | ------------------- |
| **写真が主役**      | 一覧・詳細で **装飾を最小**にし、情報は必要最低限                               | 枠・バッジ・説明文で写真が負ける    |
| **操作は軽い**      | **文字と軽いアクセント**で状態を示し、**境界は細く** | 塗りの強い操作部品の羅列、太枠・多重カード |
| **固定 UI の縦幅** | 画面上部などに固定する操作（検索・絞り込み等）は **コンテンツを圧迫しない**                               | 常時ヒント・見出しの重ねで操作帯が過大になる |
| **余白と階層**      | **余白で区切る**、同じ情報の **二重表示を避ける**                                      | リッチな区画枠の重ね          |


### 3.4 アクセシビリティと一貫性

- 意味のある操作・状態は **支援技術でも伝わるよう**にする（検索の進行中など、状態が分かること）。
- **色・余白・細部の見た目**の統一はプロダクト開発で一元管理し、ビジョンでは **意図**（§5）だけを固定する。

## 4. ブランドトーンと表記

> **操作・レイアウトのルール**は [§3.3](#33-ui--ux見た目と操作) と [§5](#5-ui--ux-基盤) と対で読む。

### 4.1 方針（要約）

- **文体**：です・ますを基本に、**短文**。長いビジネス敬語・説明書調は避ける。
- **人称**：「あなた」を避け、**みんな・自分の部屋** など第三者視点・名詞ベース。
- **ビジュアル**：ダーク既定・グレー階調・余白は贅沢に。派手なグラデ・多色は **アクセントに限定**。
- **機能名**：ユーザー向けは **欲しい** のように短い動詞・名詞。内部用語を表に出さない。
- **載せるの言い方**：ナビや常時表示は短く **載せる**。見出し・ログイン誘導・主要 CTA は **写真を載せる**（実際の操作と一致し、「部屋を載せる」のような比喩的な物扱いを避ける）。
- **プロフィールの公開感**：UI では **プロフィールページに表示**／**プロフィール設定** など、場所が想像できる言い方に寄せる。「みんなに見える」は曖昧になりやすいので避ける（条文・法務は正確さ優先で従来どおり）。
- **説明の量**：一般的なネットサービス利用を前提に、**UI とラベルで足りるところは説明を書かない**（使っていくうちに慣れる想定）。残すのは **法務・安全・誤解しやすい一点** など、本当に要る短文だけ。
- **シンプル**：ラベル・補足・ヒントは **足しすぎない**。迷ったら短くする（[§2](#2-プロダクト原則優先順位) 原則 4）。

### 4.2 アプリ内表記ルール


| 方針        | OK の例                       | 避けたい例                 |
| --------- | --------------------------- | --------------------- |
| 文体        | 載せました。もう一度お試しください。          | 載せた。試してみて。（常体の軽すぎる一文） |
| 短文        | ログインが必要です。                  | お手数ですが〜を重ねた長文         |
| 集合・場所     | みんなの部屋、マイページ                | あなたの〜                 |
| 載せるの長短   | 短い表示は **載せる**／説明・主要 CTA は **写真を載せる** | **部屋を載せる**（比喩が強く違和感になりやすい）、ユーザー向けで「投稿」だらけ |
| プロフィールの補足 | **プロフィール設定**、**プロフィールページに表示** | **みんなに見える**（曖昧・子ども向けに聞こえやすい） |
| コンテンツの呼び方 | **部屋**（ナビは **載せる**）         | ユーザー向けで「投稿」だらけ        |
| 家具の行（フォーム・詳細） | **家具・雑貨**                    | ユーザー向けで「アイテム」だらけ      |
| ブックマーク    | **保存**（UI・空状態）              | ブックマーク（ユーザー向け文言）      |
| アイテムリスト   | **欲しい**（オンオフ後も **同じラベル**でよい。状態は視覚で示す） | 「欲しい ✓」や「欲しいリスト」などの冗長表現 |
| トップへの戻り   | **みんなの部屋**（フッター・規約の戻りリンク） | ユーザー向けで「ホーム」だけに偏る    |
| エラー       | もう一度お試しください。                | 定型句の乱用で内容がぼやける        |
| 法務ページ     | 本文は正確さ優先。冒頭だけプロダクトの温度感      | 条文全体を口語だけにする          |
| 家具の導線（コピー） | **部屋・ムードを先**に。家具は **商品ページのURL／ショップのページ** など事実寄り | ヒーローや一覧で **購入先** を前面に連呼 |

### 4.3 表層コピーの優先順（部屋とムードを先に）

- **① 表（ヒーロー・フィード・ナビ）**：部屋・ムード・発見を先に。**「購入先」は前面に出しすぎない**。
- **② 補助（詳細・フォーム・ダッシュボード）**：家具・雑貨は **商品ページURL／ショップのページ** とし、辿り方は短文で足す。
- **③ 法務・規約**：正確さ優先。**購入リンク** など定型的な用語は条文に残してよい。

## 5. UI / UX 基盤

### 5.1 三層モデル（優先度のイメージ）


| 層          | 役割                     | トーン                                              |
| ---------- | ---------------------- | ------------------------------------------------ |
| **コンテンツ層** | 写真の一覧・拡大表示、家具と購入先の提示   | **部屋のムード・写真が主役**。家具と購入先は同じ流れで。装飾・オーバーレイは最小限。         |
| **操作層**    | ナビ、検索、分類、フィードの切り替えなど | **線と文字を基調**にし、画面上部など **固定する操作域の高さは抑える**。               |
| **入口層**    | 未ログイン向けの紹介・ショーケース     | 温度感は出してよい。**セール煽り・バナーだらけは NG**。 |


*操作層がコンテンツ層を食わない*ことが、[原則 1](#2-プロダクト原則優先順位) と [§3.3 の表](#3-迷ったときの判断基準) と一致する。

### 5.2 ルール（チェックリストの補足）

- **操作はコンテンツに勝たない**：ナビ・検索・絞り込みなどは **情報量と装飾を抑え**、発見・発信・購入導線のうち **画とムードが主役**になるようにする。境界は細く、**余白で階層**をつける。細部のレイアウトや部品の形は **都度最適化**してよい。
- **検索・分類の出し方**：キーワードと属性（カテゴリ・スタイル等）の **優先順位や固定領域の付け方**は固定仕様として本書に書かない。[原則 1](#2-プロダクト原則優先順位)（発見の速さ）と [原則 4](#2-プロダクト原則優先順位)（シンプル）、[§3.3](#3-迷ったときの判断基準)（コンテンツの圧迫を避ける）を満たすこと。
- **画面のムード**は [§6](#6-ターゲットと世界観) の世界観と揃え、**煽り・セール調**は避ける（§6.3）。背景・カード・見出しの **具体的なビジュアル手法は本書に載せない**。

### 5.3 ビジュアルと細部の委ね方

**名称・見た目・部品の組み立て**はプロダクト開発に委ねる。ビジョンでは §5.1〜5.2 の **意図**（写真主役・操作の軽さ・§6 との整合）だけを固定し、改善や実験で表現を変えてよい。一貫性が必要なときは **トークンやスタイルの体系で表現を一本化**してよい。

## 6. ターゲットと世界観

**§6 の読み順**：**6.1** で人物像とライフステージの置き方 → **6.2** でキーワード・エリアの語彙をカテゴリ別に一覧（コピー・発見の表現と揃える **基準**）→ **6.3** でトーンと「コンクリート感」→ **6.4** でエリア名の五十音早見（**6.2 と同一組**。一覧の参照用）。

### 6.1 メインターゲット（中心の人物像）

> **排他ではない**が、コピー・分類の語彙の **一番の寄せ先** として明示する。

- **住まい**：**賃貸**を第一想定（持ち家・シェアも歓迎。**賃貸以外を排除する文言・アルゴリズムは NG**）。**一人暮らし・ワンルーム** を日常的なスケールの前提にしやすい。
- **年齢感**：**大学生〜20代後半**を中心に、「自分の部屋を育てる」層。**社会人**は少しずつ理想に近づける **積み上げストーリー**（後述）も想定する。
- **ムードの芯（想像の軸）**：**男の一人暮らし**を、プロダクトを考えるときの **一人称の寄せ先** にしてよい（制作者の体感と一致させ、ブレを減らす）。**ファッションにも気を遣うおしゃれ男子**の **モノトーン／ハイセンス** な部屋。**ガジェット好き・こだわり・丁寧に生きる** などの価値観と相性がよい。
- **ライフステージ別の家具ストーリー（価値の置き方）**
  - **大学生など／予算が限られる段階**：**高い家具一択**ではなく、**ニトリ・IKEA など手の届く店でも「ちゃんと揃えた」** と感じられる **入り口** になればよい（見た目・組み合わせ・リンクの並び方で学べる）。
  - **社会人など／少しずつ投資できる段階**：**高いものと安いものを組み合わせながら**、理想の部屋を **時間をかけて構築するストーリー** を肯定する（一発で完成させるプレッシャーはかけない）。**少し高めの家具**のイメージとしては、服のセレクトと世界観がつながっている **Journal Standard Furniture（ジャーナルスタンダード ファニチャー）** のような **ミドル〜セレクト寄りのライン**が典型例のひとつ（**提携・推奨の意味ではない**。あくまでペルソナの価格帯・ムードの参照）。
- **憧れのイメージ**：無機質なベースに **コーヒー・光・植物** などで体温を足す。**サンプルルームより住んでいそうな画**。
- **参照ムード（外部）**：[gen_kurasi（Instagram）](https://www.instagram.com/gen_kurasi/) のような **グレー階調・小物・光**（公式提携ではない）。

#### 男性に「寄せる」ときと、マーケットを狭めない話

- **寄せてよいもの**：内的な **ペルソナ・コピーのトーン・例示写真のイメージ・分類・コピー上の優先**（「誰のためのサービスか」をチームで共有するため）。
- **狭めないもの**：**利用条件・コピーでの明示的排除**（「男性専用」など）は **しない**。同じムードを好む **女性・その他の性別**も歓迎し、**アルゴリズムやコミュニティで女性を排除する設計は NG**（[原則・やらないこと](#7-やらないこと) の「特定スタイルの排他」と同型）。
- **TAM の感覚**：想像の軸を **男の一人暮らし**に置いても、**実際の利用者はより広くなりうる**（好みは性別で二分されない）。狭まるのは **「男性しか来るな」という文言・演出** を前面に出したときであり、**ペルソナを内部ドキュメントとして持つこと自体**は、むしろメッセージの一貫性に効く。

### 6.3 若者向け・おしゃれ・「コンクリート感」

- **スマホでさっと**：スクロールで発見、**文章は少なくてよい**（ただしトーンは §4）。
- **リアル志向**：「住んでる感」「賃貸でもここまで」のほうが信頼されやすい。
- **行動は軽く**：いいね・保存・欲しいのハードルは低く。
- **おしゃれ**：フィードの気分。閉店感・高飛車さ・チラシバナー・過剰セール煽りは NG。
- **コンクリート感**：**素材指定ではなくムード**。UI は無機質寄りでも、**生活の温度は写真に残す**。「コンクリ部屋だけが正解」ではない。

## 7. やらないこと

例外を出すなら **本書と改訂履歴に理由** を残す。

- **総合 EC の完全代替**（在庫・決済・配送完結を主目的にしない）
- **家具百科**（網羅より **好みの部屋に出会う体験**）
- **過度なゲーミフィケーション**（バッジだらけ、ランキング至上）
- **特定スタイルの排他**（中心の好みはあっても **排除しない**）
