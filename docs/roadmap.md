## 目的

チーム内で「今どこを先に詰めるか」を揃えるためのメモです。  
機能の網羅表ではなく、**現在地 → 次に伸ばすテーマ → 後ろに置くもの** を見やすくすることを目的にします。

## 現在地（2026-03-21 時点）

すでにある土台:

- みんなの部屋のフィード、検索、カテゴリ、スタイルタグ、価格帯による発見導線
- 部屋詳細、家具・雑貨一覧、商品ページ URL、関連部屋、欲しい、保存、いいね
- 写真を載せる流れ、プロフィール、フォロー、ダッシュボード
- ダーク既定の UI と、若年層向けのトーンの基礎

まだ薄い領域:

- 発見の精度と再訪導線の深さ
- 購入リンクの信頼表示と横断性
- 発信を支える軽い補助
- 賃貸リアルの文脈を発見に変える仕組み

## 優先の軸

迷ったら [product-vision.md §2](./product-vision.md#2-プロダクト原則優先順位) の順で譲りにくいです（**発見 → 発信 → 購入セット → シンプル**）。

1. 若者のおしゃれ・ムードを**発見**できるか（速く辿り着けるか含む）
2. **自分のおしゃれを発信**し、続けられるか
3. 写真と「どこで買えるか」が**セット**か（①②を弱めない）
4. **シンプル**か

## ロードマップの見方

- **Now**: 今の土台の上に最も乗せやすく、[§2](./product-vision.md#2-プロダクト原則優先順位) の原則 1〜4 に直結するもの
- **Next**: 効果は大きいが、設計・運用・モデレーションの見積りが必要なもの
- **Later / Experiments**: 検証価値はあるが、説明責任やコストに注意が必要なもの

## Now

### 1. 発見の質を上げる

狙い:
好みの部屋に辿り着くまでの手数を減らし、保存や欲しいからの再訪も自然につなげる。

テーマ:

- 関連部屋・おすすめの精度改善
- フィードミックスの調整
- 保存・欲しい起点の再訪導線
- 同じ購入 URL を軸にした横断発見

成功の見え方:

- みんなの部屋から詳細への到達が増える
- 保存や欲しい経由の再訪が増える
- フィードが単調に見えにくくなる

### 2. 写真 × 購入先の信頼を厚くする

狙い:
「このリンクは信用してよいか」を短い情報で判断できるようにする。

テーマ:

- リンク最終確認日や弱い状態表示
- 購入リンクの関係性ラベル
- 家具・雑貨メタの最小追加
- 欲しいの共有や比較しやすさの改善

成功の見え方:

- 詳細ページでの外部遷移の迷いが減る
- リンクまわりの透明性に関する説明コストが下がる

### 3. 載せる摩擦を下げる

狙い:
リンク付き UGC を増やすために、写真を載せる側の途中離脱を減らす。

テーマ:

- 下書きの同期
- 複数枚アップロードの並び替え / 一括編集
- URL 貼り付けの補助
- 載せたあとの軽いインサイト

成功の見え方:

- 写真を載せる完了率が上がる
- 載せ続ける人が増える
- 部屋に含まれる家具・雑貨情報が安定する

### 4. 賃貸リアルを発見導線に変える

狙い:
NOOK らしさである「住んでいる部屋のリアルさ」を、検索や関連導線にも効く資産にする。

テーマ:

- 間取り・制約のミニタグ
- 部屋単位の予算ムード
- 同じ文脈を持つ部屋の束ね方

成功の見え方:

- サンプルルーム的な写真との差が伝わる
- ユーザーが自分ごと化しやすくなる

## Next

### 1. 再訪チャネルの拡張

- PWA
- プッシュ通知
- フォローや保存に紐づく穏やかなリマインド

条件:
通知や再訪導線がセール調にならず、若年層向けトーンを崩さないこと。

### 2. 認証と基盤の拡張

- メールリンクなどのログイン追加
- 下書きや同期機能を支えるデータ基盤
- アクセシビリティの継続投資

条件:
スパム対策、運用負荷、説明コストを先に見積もること。

### 3. ソーシャル最小

- コメント
- 定型リアクション
- 引用シェア

条件:
写真主役を壊さず、モデレーションを回せる最小単位から始めること。

### 4. 差別化を太くする重めの仕組み

- 部屋の変遷タイムライン
- 代替候補 URL
- 店舗向け閲覧モード
- 縦切りコレクション

条件:
価値は大きいが、情報構造が一段複雑になるため、実装前に設計を整理すること。
