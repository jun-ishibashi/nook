# NOOK ロードマップ（優先度の仮説）

> **確約ではない。** 並びと内容は学びで変える。ブレない軸は常に [`product-vision.md`](product-vision.md) の [§2 プロダクト原則](product-vision.md#2-プロダクト原則優先順位)・[§7 やらないこと](product-vision.md#7-やらないこと) を正とする。  
> **実装の正**は [`README.md`](../README.md) の「機能」とコード。ここでは「その土台の上で、次に厚くするテーマ」の順序だけを書く。

## 目的

チーム内で「いま何を先に詰めるか」を揃えるための **作業仮説**。[候補の全体リスト](product-vision.md#8-将来検討できる機能候補)はビジョン §8 にあり、ロードマップはそこから **今の仮説で選んだ並び** に過ぎない。

## 優先の軸

並び替えに迷ったら、次の順で譲りにくい（[§2 詳細](product-vision.md#2-プロダクト原則優先順位)）。

1. 写真と「どこで買えるか」がセット  
2. 発見が速い  
3. 恥ずかしくないトーン  
4. 載せる側が続けやすい  

## GTM・収益との整合

[市場・GTM・マネタイズ](product-vision.md#市場における立ち位置と差別化) のデフォルトどおり、**供給（UGC）と再訪が先**、収益の具体化はモートが薄いうちは後ろに寄せる。[マネタイズの設計方針](product-vision.md#マネタイズの設計方針中長期) の判断表を満たさない実験はしない。

## Near（いま厚くしやすい）

コア価値（原則 1〜4）に直結し、§7 に触れにくいものを前に。

- **発見の質** … おすすめ／関連の強化、フィードのミックス、保存・欲しいからの再訪導線（通知はセールだらけにしない）など（[§8](product-vision.md#8-将来検討できる機能候補) の **8.1**）
- **写真 × 購入先の信頼** … 家具行の任意メタ（価格帯・ブランド正規化は百科化しない）、リンクの状態・ドメインの弱い表示、欲しいのエクスポート／共有の法務配慮など（[§8](product-vision.md#8-将来検討できる機能候補) の **8.2**）
- **載せる摩擦の削減** … 下書きのクラウド同期、複数枚の並び替え・一括メタ編集、投稿後の軽いインサイト（バッジ至上にしない）など（[§8](product-vision.md#8-将来検討できる機能候補) の **8.3**）

## Mid（境界が太い・設計が要る）

- **ログインの幅** … メールリンク等の追加（スパム対策・トーンは §4）（[§8](product-vision.md#8-将来検討できる機能候補) の **8.5**）
- **再訪チャネル** … PWA・プッシュ（頻度・文面は [§6.3](product-vision.md#63-若者向けおしゃれコンクリート感) に反しない）（[§8](product-vision.md#8-将来検討できる機能候補) の **8.5**）
- **アクセシビリティの継続投資**（[§8](product-vision.md#8-将来検討できる機能候補) の **8.5**）
- **コミュニケーション最小** … コメント／リアクション、部屋の引用シェア（モデレーション・写真主役）（[§8](product-vision.md#8-将来検討できる機能候補) の **8.4**）

## Later / 実験枠

## §8 との関係

§8 は **着手前に原則を通す候補の倉庫**。ロードマップは、その倉庫から取り出した項目の **現在の順序のメモ** にすぎない。項目の追加・削除は §8 を先に更新するか、§8 に無い実験なら改訂履歴か下表に理由を残す。

## スコープの縁取り（§7 との整合）
